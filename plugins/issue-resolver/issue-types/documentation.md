# Documentation Update Procedure

## Overview

This document provides the procedure for updating documentation.

## Resolution Steps

### 1. Understand Documentation Needs
- Read the issue to understand what documentation is needed
- Identify which documents need to be created or updated
- Clarify the target audience and purpose
- Understand the scope of documentation work

### 2. Gather Information
- Review the code or features being documented
- Check existing documentation for gaps or outdated content
- Understand the technical details that need to be explained
- Identify any examples or use cases to include

### 3. Plan Documentation Structure
- Determine the organization and flow of information
- Plan sections and headings
- Consider what examples or diagrams might help
- Follow existing documentation patterns in the project

### 4. Write Documentation
- Use clear, concise language
- Provide accurate technical information
- Include practical examples where helpful
- Follow the project's documentation style and format
- Ensure completeness without unnecessary verbosity

### 5. Review for Quality
- Check for accuracy of technical details
- Verify code examples work correctly
- Ensure explanations are clear and understandable
- Check for typos and grammatical errors
- Confirm all requested documentation is covered

### 6. Create Pull Request
- Use `gh pr create` command to create the PR
- Title the PR clearly (e.g., "Docs: [brief description]")
- Include in PR description (in Japanese):
  - 概要: What documentation was updated
  - 変更の背景: Why this documentation was needed
  - 変更詳細: What information was added/updated
- Reference the original issue number

## Important Notes

- Focus on clarity and accuracy
- Keep documentation up to date with current code
- Use consistent terminology throughout
- Include practical examples when helpful
- Consider the target audience's knowledge level
