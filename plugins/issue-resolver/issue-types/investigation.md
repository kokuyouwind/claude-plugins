# Investigation Task Procedure

## Overview

This document provides the procedure for investigation and analysis tasks.

## Resolution Steps

### 1. Clarify Investigation Scope
- Read the issue carefully to understand what needs to be investigated
- Identify specific questions to be answered
- Determine the scope and boundaries of the investigation
- Clarify expected deliverables

### 2. Gather Information
- Search for relevant code using appropriate tools
- Read documentation and comments
- Review git history and past changes if relevant
- Examine related issues and PRs

### 3. Conduct Analysis
- Analyze the gathered information systematically
- Document findings as you discover them
- Use the debugging process guidelines for structured analysis
- Track all relevant file paths and line numbers

### 4. Organize Results
- Structure findings in a clear, logical manner
- Separate facts from interpretations
- Highlight key insights and conclusions
- Provide evidence for claims (file paths, code snippets, etc.)

### 5. Report Findings
- Post investigation results as a comment on the issue
- Use clear, concise language in Japanese
- Include:
  - 調査対象: What was investigated
  - 調査方法: How the investigation was conducted
  - 発見事項: Key findings
  - 結論: Conclusions and recommendations
- Reference specific code locations with file:line format

## Important Notes

- Focus on answering the specific questions asked
- Provide evidence-based conclusions
- Don't implement changes unless explicitly requested
- If investigation reveals issues, suggest next steps
- Be thorough but concise in reporting
