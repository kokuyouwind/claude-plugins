# Feature Addition Procedure

Follow these steps to add new features:

## 1. Understand Requirements

- Read the issue description to understand the requested feature
- Clarify the feature scope and boundaries
- Identify any constraints or requirements

## 2. Design Approach

- Consider implementation options
- Present multiple approaches if there are trade-offs
- Choose an approach that aligns with existing codebase patterns
- Keep the design simple and focused

## 3. Implement Feature

- Follow existing code patterns and conventions
- Write clean, maintainable code
- Avoid adding unnecessary abstractions or over-engineering
- Focus on the requested functionality only

## 4. Create Pull Request

- Use `gh pr create` to create the PR
- Write PR description in Japanese
- Include:
  - Feature overview and purpose
  - Implementation approach
  - Usage examples or testing notes (if applicable)
- Reference the issue in the PR description with "Closes #<issue-number>"
