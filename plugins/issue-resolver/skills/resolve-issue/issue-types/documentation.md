# Documentation Update Procedure

Follow these steps to update documentation:

## 1. Identify Documentation Needs

- Read the issue to understand what documentation needs updating
- Locate the relevant documentation files
- Understand the current state and gaps

## 2. Gather Information

- Collect accurate and up-to-date information
- Review code and features being documented
- Verify technical details
- Check for related documentation that may need updates

## 3. Update Documentation

- Make necessary updates to documentation files
- Ensure accuracy and completeness
- Use clear, concise language
- Follow existing documentation style and format
- Add examples if helpful

## 4. Review Quality

- Verify technical accuracy
- Check for clarity and readability
- Ensure formatting is consistent
- Test any code examples if included

## 5. Create Pull Request

- Use `gh pr create` to create the PR
- Write PR description in Japanese
- Include:
  - Summary of documentation changes
  - Reason for updates
  - Areas affected
- Reference the issue in the PR description with "Closes #<issue-number>"
