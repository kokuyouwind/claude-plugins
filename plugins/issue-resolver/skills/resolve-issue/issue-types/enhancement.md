# Enhancement Procedure

Follow these steps to enhance existing features:

## 1. Understand Enhancement Request

- Read the issue to understand what needs to be improved
- Identify the existing feature being enhanced
- Clarify the enhancement scope and goals

## 2. Design Enhancement

- Review the current implementation
- Consider multiple enhancement approaches if applicable
- Present options if there are significant trade-offs
- Choose an approach consistent with existing patterns
- Keep the enhancement focused and minimal

## 3. Implement Enhancement

- Enhance the existing feature as requested
- Follow existing code patterns and conventions
- Maintain backward compatibility if applicable
- Avoid unnecessary refactoring or feature additions
- Write clean, maintainable code

## 4. Create Pull Request

- Use `gh pr create` to create the PR
- Write PR description in Japanese
- Include:
  - Enhancement overview and purpose
  - Implementation approach
  - Impact on existing functionality
  - Testing notes (if applicable)
- Reference the issue in the PR description with "Closes #<issue-number>"
