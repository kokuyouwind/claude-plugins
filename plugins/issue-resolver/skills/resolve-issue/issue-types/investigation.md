# Investigation Task Procedure

Follow these steps to complete investigation tasks:

## 1. Define Investigation Scope

- Read the issue to understand what needs to be investigated
- Clarify the investigation objectives
- Identify key questions to answer

## 2. Conduct Investigation

- Search relevant code sections
- Review documentation and commit history
- Analyze data and patterns
- Test hypotheses if applicable
- Document findings as you go

## 3. Organize Results

- Structure findings clearly and logically
- Highlight key insights and conclusions
- Include code references with file paths and line numbers
- Note any limitations or uncertainties

## 4. Report Findings

- Post investigation results as an issue comment
- Write the comment in Japanese
- Include:
  - Investigation summary
  - Detailed findings
  - Conclusions and recommendations (if applicable)
  - Suggested next steps (if applicable)
- Do not create a PR for investigation-only tasks unless implementation is also requested
