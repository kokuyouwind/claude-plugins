# type:investigation - Investigation Task Procedures

## Overview

Detailed procedures for investigating and analyzing codebase or systems.

## Implementation Steps

### 1. Clarify investigation target

- Review investigation purpose described in the Issue
- Clarify what to investigate and what information is needed
- Identify investigation scope (files, directories, system components, etc.)

### 2. Investigate related code, documentation, and history

- Read related source code
- Understand change background from commit and PR history
- Review existing documentation and comments
- Reference external resources (library documentation, etc.) as needed

### 3. Organize investigation findings

- Structure and organize discovered facts
- Highlight key findings
- Visualize with diagrams or tables as needed
- Summarize conclusions and recommendations

### 4. Post concise investigation summary as Issue comment

- Write findings in clear, readable format
- Recommended structure:
  - **Investigation Overview**: What was investigated
  - **Key Findings**: Main discoveries
  - **Details**: Specific investigation results
  - **Conclusion**: Conclusions and recommendations
- Include file paths and line numbers for code references
- If too long, focus on key points; detailed findings can be a separate document

## Notes

- Investigation tasks typically don't require code changes or PR creation
- Suggest creating new Issues if findings warrant action
- Clearly distinguish facts from speculation
- Report minor issues found during investigation as separate Issues if needed
